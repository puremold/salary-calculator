# 계산모아 — 배포 & 수익화 가이드

연봉 실수령액 계산기(프로그래매틱 SEO 사이트)입니다.
연봉마다 자동으로 검색 최적화 페이지가 생성되어, 구글 검색을 통해 방문자가 들어오고
나중에 **애드센스 광고**로 수익이 발생하는 구조입니다.

운영비 **0원**(Vercel 무료 티어), AI API 사용 **없음** → 변동비 손실 구조적으로 없음.

---

## 한눈에 보는 전체 흐름

```
①코드 → ②GitHub → ③Vercel 배포 → ④구글 서치콘솔(색인) → (트래픽 쌓임) → ⑤애드센스 수익
```

핵심은 ④번입니다. 구글이 페이지 100여 개를 색인해야 검색에 노출됩니다(보통 2~6주).

---

## 1단계. GitHub에 코드 올리기 (무료, 약 5분)

1. https://github.com 가입 → 로그인
2. 우측 상단 **+ → New repository**
   - Repository name: `salary-calculator` (아무거나)
   - **Public** 선택, README 등 체크는 모두 해제
   - **Create repository** 클릭
3. 만들어진 페이지에 나오는 주소(`https://github.com/내아이디/salary-calculator.git`)를 복사
4. 이 폴더에서 아래를 실행 (PowerShell):

```powershell
cd "C:\Users\hufst\Desktop\for claude\salary-calculator"
git remote add origin https://github.com/내아이디/salary-calculator.git
git branch -M main
git push -u origin main
```

> 처음 push 시 GitHub 로그인 창이 뜨면 브라우저로 인증하면 됩니다.

---

## 2단계. Vercel로 배포 (무료, 약 3분)

1. https://vercel.com → **Continue with GitHub**로 가입
2. **Add New… → Project**
3. 방금 만든 `salary-calculator` 저장소 **Import**
4. 설정은 기본값 그대로 → **Deploy** 클릭
5. 1~2분 뒤 `https://salary-calculator-xxxx.vercel.app` 주소가 나옵니다. 접속해서 확인!

### 배포 후 꼭 할 것 — 사이트 주소 등록

1. Vercel 프로젝트 → **Settings → Environment Variables**
2. 추가:
   - Name: `NEXT_PUBLIC_SITE_URL`
   - Value: 위에서 받은 실제 주소 (예: `https://salary-calculator-xxxx.vercel.app`, 끝에 `/` 없이)
3. **Deployments → 최신 배포 → ⋯ → Redeploy** (환경변수 반영)

> 이 주소가 sitemap·canonical·구조화데이터에 들어가므로 SEO에 중요합니다.

---

## 3단계. 구글 검색에 등록 (가장 중요, 약 10분)

1. https://search.google.com/search-console 접속
2. 속성 추가 → **URL 접두어** → 사이트 주소 입력
3. 소유권 확인:
   - 가장 쉬운 방법: **HTML 태그** 방식 선택 → `<meta name="google-site-verification" ...>` 값 복사
   - `src/app/layout.tsx`의 `metadata`에 아래 한 줄 추가 후 다시 push:
     ```ts
     verification: { google: "복사한_값" },
     ```
   - 또는 Vercel을 도메인 연결했다면 DNS 방식도 가능
4. 확인 완료 후 좌측 **Sitemaps** → `sitemap.xml` 입력 → 제출
5. 끝! 이제 구글이 109개 페이지를 차례로 색인합니다 (2~6주 소요).

추가로 https://www.bing.com/webmasters 에도 같은 방식으로 등록하면 좋습니다(네이버 유입 일부 포함).

---

## 4단계. 애드센스로 수익화 (트래픽이 쌓인 뒤)

> 애드센스는 어느 정도 콘텐츠/방문자가 있어야 승인됩니다. 보통 색인 후 방문자가
> 생기기 시작하면 신청하세요.

1. https://adsense.google.com 가입 → 사이트 주소 등록 → 심사(수일~수주)
2. 승인되면 게시자 ID(`ca-pub-XXXX...`) 발급
3. Vercel → Settings → Environment Variables 에 추가:
   - `NEXT_PUBLIC_ADSENSE_CLIENT` = `ca-pub-XXXXXXXXXXXXXXXX`
4. Redeploy → 광고 스크립트가 자동으로 삽입됩니다.
   (개별 광고 단위 배치는 이후 `src/components`에 `<ins class="adsbygoogle">` 추가로 확장)

## 5단계. 성장시키기 (이 사이트가 돈을 버는 진짜 엔진)

검색 트래픽 = 페이지 수 × 검색 수요. 페이지를 **레고처럼 추가**하면 트래픽이 커집니다.
다음 계산기들은 같은 구조로 쉽게 붙일 수 있습니다(클로드에게 요청하세요):

- 시급 → 월급 계산기 / 주휴수당 계산기
- 퇴직금 계산기
- 자동차세 계산기 (배기량별 자동 페이지)
- 부동산 중개수수료(복비) 계산기 (거래가별 자동 페이지)
- 실업급여 계산기

각 계산기마다 수백 개 페이지가 더 생기고 → 색인 → 트래픽 → 광고 수익으로 이어집니다.

---

## 현실적인 기대치

- **1개월차**: 배포 + 색인 시작. 방문자 0~수십, 수익 거의 0. (정상입니다)
- **2~3개월차**: 일부 키워드 노출 시작, 일 방문자 수십~수백.
- **4~6개월차**: 누적 색인·순위 상승. 트래픽이 광고 수익(월 수만~수십만 원)으로 전환되기 시작.
- 월 100만 원은 페이지/주제를 꾸준히 늘리고 색인이 쌓인 결과로 도달하는 목표입니다.

> SEO는 복리입니다. 초기엔 느리지만 페이지와 시간이 쌓일수록 가속됩니다.
